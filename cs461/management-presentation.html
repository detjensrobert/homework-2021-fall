Fun With Kubernetes

K8S Cluster Deployment

Robert Detjens

October 28, 2021

Project Summary

  • Kubernetes Cluster on Raspberry Pi
  • Deployment pipeline for containerized applications
  • Web API for project management

Goals

  • Make it easy for developers to deploy containerized applications to a K8S cluster

Requirements

  1. Kubernetes Setup
  2. Project Config File
  3. Github Container Build Pipeline
  4. Kubernetes Deployment Pipeline
  5. Deployment UI Website
  6. Final Integration
  7. Beta Testing

Deliverables

1. Kubernetes Setup

  1. Install Kubernetes on the Raspberry Pi (1-2 days)
  2. Configure Kuberentes on the Raspberry Pi (1-2 days)
  3. Connect to the Raspberry Pi Admin tools with K8S Lens (3 Days)
  4. Test Kubernetes on the Raspberry Pi (3 days)
  5. Code reviews and release (4-5 Days)

2. Kubernetes CI/CD Config File Integration

  1. Research what settings are needed (2-3 days)
  2. Work with Deployment UI Website team to define template format (4-6 days)
  3. Create end-user documentation for config template (1 days)

3. Github Container Pipeline

  1. Create build script to pull repo from user-provided repository (1 day)
  2. Modify build script to build container image (1-3 days)
  3. Coordinate with K8S Deployment team to define container handoff API (2-3 days)
  4. Modify build script to send built image to K8S onboarding (1-2 days)
  5. Modify script to ensure clean building environment (1-2 days)
  6. Code Reviews and release (4-5 Days)

4. Kubernetes Deployment Pipeline

  1. Watch the images in the deployment area (1-2 days)
  2. Deploy a new image in the deployment area (2-3 Days)
  3. Coordinate with Github Container Build team to define container handoff API (2-3 Days)
  4. Testing (2-3 Days)
  5. Code Reviews and release (4-5 Days)

5. User-facing Deployment UI Website

6. Final Integration

7. Beta Testing
